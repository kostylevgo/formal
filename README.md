## Практическая задача из домашнего задания №3
Для детерминизации автомата введите данные недетерминированного автомата в файл `task-deterministic.rs` и запустите:
```
cargo run --bin task-deterministic
```
В потоке вывода вы увидите всю информацию о детерминированном автомате: количество состояний, номера стартового и завершающих состояний и все его переходы. Чтобы проверить, распознаётся ли слово автоматом, введите запрос в тот же файл после детерминизации автомата.

*P.S. В данный момент ввод автомата из файла не поддерживается, программа не соответствует требованиям к заданию, проверять её не надо.*

## Алгоритм Эрли
Для запуска введите
```
cargo run --bin task-earley
```
Мои тесты можно запустить командой
```
cargo test
```
Проверяющие могут запустить свои тесты через stdin:
```
cargo run --bin task-earley <test.txt
```
Если входные данные не соответствуют формату входных данных, выполнение программы будет прервано с соответствующим сообщением об ошибке. Исключение: ошибка в $N$ или $\Sigma$.

## Покрытие
Для запуска покрытия установите
```
cargo install cargo-llvm-cov
```
и запустите
```
cargo llvm-cov --html --ignore-filename-regex "task-"
```
Результаты можно будет увидеть в `./target/llvm-cov/html`. В задании на регулярные выражения Line Coverage составляет 95%. Покрыты все функции, кроме некоторых функций вывода автоматов в поток вывода и строк, обрабатывающих ошибки, возникающие при этом выводе.
